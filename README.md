# FaculRide-Mobile
Projeto Integrador para dispositivos mobiles

## ✅  Requisitos Funcionais 

| Nº Requisito Funcional | Nome                 | Descrição                                                                 |
|------------------------|----------------------|----------------------------------------------------------------------------|
| RF001                  | Login                | Fazer Login com usuário e senha                                           |
| RF002                  | Login com Biometria  | Fazer Login com biometria                                                 |
| RF003                  | Gerenciar Usuário    | Consultar, cadastrar, alterar ou excluir usuário do site                  |
| RF004                  | Chamar Carona        | Solicitar, passageiro solicita a sua carona ao motorista                  |
| RF005                  | Oferecer Carona      | Oferecer carona, motorista disponibiliza a sua carona ao passageiro      |
| RF006                  | Procurar Carona      | Pesquisar e visualizar lista de motoristas disponíveis                    |
| RF007                  | Geolocalização       | Rastrear localização dos passageiros e motoristas                         |
| RF008                  | Notificações         | Enviar notificações via e-mail, push de notificação ou pelo site para informar os usuários |
| RF009                  | Avaliação            | Implementar um sistema de avaliação de 1 a 5 para os usuários             |
| RF010                  | Comentários          | Comentários relacionados aos usuários                                     |

## 🚫 Requisitos Não Funcionais

| Nº Requisito Não Funcional | Nome                      | Descrição                                                                 |
|----------------------------|---------------------------|----------------------------------------------------------------------------|
| RNF001                     | Banco de dados            | Utilização de Banco de dados SQL Server                                   |
| RNF002                     | React Native              | Utilizar a biblioteca Javascript React Native                             |
| RNF003                     | Segurança                 | O sistema deve ser protegido contra acessos não autorizados               |
| RNF004                     | Escalabilidade            | O sistema deve ser capaz de aumentar ou diminuir sua capacidade           |
| RNF005                     | Velocidade e Eficiência   | O sistema deve responder rápido, no máximo 5 segundos, para aplicativos mobiles, além do back funcionar rápido também |
| RNF006                     | Disponibilidade           | O sistema deve estar disponível 24h por dia, 7 dias por semana            |
| RNF007                     | Usabilidade               | Interface intuitiva e fácil de usar, mesmo para iniciantes                |
| RNF008                     | Confiabilidade            | O sistema deve ser robusto e resistente a falhas                          |
| RNF009                     | Portabilidade             | Deve funcionar em diversos dispositivos mobiles e Sistemas Operacionais   |
| RNF010                     | Manutenibilidade          | Código organizado e documentado para facilitar atualizações futuras       |
| RNF011                     | Baixo consumo de bateria  | O sistema deve consumir 17% por hora a bateria                      |
| RNF012                     | Baixo consumo de dados    | O app deve consumir 4MB por hora de dados móveis                 |
| RNF013                     | Tamanho Reduzido          | O app deve ter um tamanho de armazenamento no máximo 50MB                          |  
| RNF014                     | Armazenamento de Imagens  | O app irá utilziar o serviço S3 da AWS para armazenar as imagens de perfil e CNH dos usuários |
